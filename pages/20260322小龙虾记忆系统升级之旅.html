第四阶段：架构优化 🏗️

五层文件记忆架构

  • L1 SESSION-STATE — 热 RAM（当前会话）
  • L2 Git-Notes — 冷存储（每日日志）
  • L3 MEMORY.md — 精选归档（长期结论）
  • L4 memory/ — 每日日志（原始记录）
  • L5 自学习 — 经验总结（避免踩坑）

唤醒方式

分层检索，按需激活

第五&六阶段：激活模型 & 伤疤记忆 🎯

同心圆激活模型

  • L1（当前会话）— 优先检索，80% 命中率
  • L2（高频决策）— 次级检索，15% 命中率
  • L3（完整日志）— 深度检索，5% 命中率

伤疤记忆库（5个踩过的坑）

  • 作者姓名拼写错误 — 多源验证优先
  • 飞书文档创建不完整 — 创建后验证
  • API 频率限制 — 使用后台任务
